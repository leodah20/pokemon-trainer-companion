Pokemon Trainer Companion

A real-time, AI-powered overlay for Pokémon GO — read a screen live, ground a Gemini answer in it, and hand back an actual tip. Everything else in the repo is the foundation that sits under it.

v1.0 Beta — closed testing RN 0.86 · bare workflow NestJS 11 + Prisma 7 Kotlin native overlay MIT licensed

Flagship overlay — native files

The only Kotlin in the repo, and the project's actual differentiator.

OverlayModule.kt

React Native bridge. hasOverlayPermission, requestOverlayPermission, showOverlay, hideOverlay, updateOverlayText, requestScreenCapturePermission, startLiveCapture, captureLiveFrame, stopLiveCapture. Emits PTCOverlayTapped and PTCOverlayFrameText via a static reactApplicationContextHolder — so the service can reach JS without an Activity reference.

ScreenCaptureService.kt

Foreground service (foregroundServiceType=mediaProjection). Opens the real MediaProjection session, a VirtualDisplay + ImageReader for frames, and runs its own native polling loop — grabs a frame, runs ML Kit OCR directly in Kotlin, emits recognized text. Native specifically because a JS-side setInterval doing the same thing stalled once the Activity lost foreground.

overlayBridge.ts

mobile/src/data/overlay/overlayBridge.ts — thin JS wrapper; every function is a safe no-op on iOS or if the native module isn't linked.

Flagship overlay — capture flow

  1. requestOverlayPermission() — trainer grants SYSTEM_ALERT_WINDOW via the system settings screen
  2. requestScreenCapturePermission() — Android's real MediaProjection consent dialog
  3. startLiveCapture() — starts ScreenCaptureService, opens the MediaProjection session, begins native polling
  4. JS listens via onOverlayFrameText(text) — analyzeOcrText(text) runs species lookup + IV/PvP/bulk analysis, the same pipeline the gallery-picker flow uses
  5. updateOverlayText(body) pushes a formatted result — species, CP, IV, best PvP league, bulk tier, tip — into the floating window
  6. On the first detection of a new species, an async fetchCompanionSuggestion() call upgrades the rule-based tip to a Gemini one — once per species, not every poll tick, to bound cost and latency
  7. Tapping the overlay brings PTC to the foreground and emits PTCOverlayTapped, which OverlayDemoScreen catches and navigates to Professor Mode

Flagship overlay — gotchas already paid for

Wrong-thread crash Native module methods run on RN's own native-modules thread, not the true Android UI thread. The overlay view is attached from whichever thread calls showOverlay() — any later mutation from a different thread throws CalledFromWrongThreadException. Fix: every WindowManager/view op goes through Handler(Looper.getMainLooper()).
MediaProjection callback ordering registerCallback(...) must be called before createVirtualDisplay() on newer Android versions, or it throws IllegalStateException.
ML Kit not compile-visible @react-native-ml-kit/text-recognition's own build.gradle declares ML Kit as implementation, which Gradle does not expose transitively. app/build.gradle needs its own explicit dependency for ScreenCaptureService.kt to call the API directly.
Testing "share one app" from inside that app The shared app only produces frames while it's actually in the foreground — awkward when you want to watch results from the requesting app's own UI. Share entire screen sidesteps this for manual testing.

Standing constraints for agents

→Keep the app lightweight — justify any new native dependency; prefer pure-JS libraries when one exists.
→Prefer adb install over USB for physical-device testing rather than sharing APK files.
→On a USB reconnect: check adb devices + Metro's localhost:8081/status before launching, not after a red error screen. Re-run both adb reverse tunnels every time.
→Update README's progress table and a relevant doc after every feature — not just when reminded.
→Any change under mobile/android/ needs a full Gradle rebuild + reinstall — Metro's hot-reload does not pick up native changes.
→Never claim a UI fix works without live on-device verification — this repo has a documented history of fixes that passed typecheck but failed for real.
→Google Sign-In is blocked pending the project owner creating OAuth credentials in Google Cloud Console — do not attempt this on their behalf.
→GEMINI_API_KEY lives only in backend/.env (gitignored) — never commit or expose it.
